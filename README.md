# subway

🚇 고령층 무임승차 시간 제한 정책 수립을 위한 교통 혼잡도 분석 시스템
 
본 시스템은 지하철 재정 적자 해소 및 출퇴근 시간대 혼잡도 완화를 위해 고령층 무임승차 제한 시간 대역을 과학적으로 도출하는 것을 목적으로 합니다. 역별 유동인구 패턴을 대화형 차트로 분석하여 가장 실효성 있는 요금 규제 타이밍을 제안합니다.




웹사이트 링크 [https://easy2safehome.streamlit.app/](https://subway-auc3tpflapcfs9nqeuogcl.streamlit.app/)
🤖 AI 프롬프트 링크: https://snowboard.sookmyung.ac.kr/mod/assign/view.php?id=2006011&action=view (스노보드로 연결)

📂 데이터
•	시간별혼잡도.csv: 호선, 역번호, 출발역, 5시-25시별 지하철 혼잡도 (정량 데이터)
•	노인이용.csv: 역번호, 역명, 승하차 구분, 6-25시별 우대권(노인) 이용 승객 수 (정량 데이터)
•	역세권.csv: 역번호, 역주변 주요 시설/출구 정보, 호선, 역명 (텍스트 데이터)


**데이터 클린징 과정**

•	Key 매핑 표준화: 세 데이터셋의 기준 키인 역번호를 중심으로 데이터 정렬 및 이기종 데이터(호선 정보 등) 결합.
•	시간대 정렬: '혼잡도'와 '노인 승하차 수'의 시간대 정의를 기존 30분 단위와 1시간 단위가 혼재하므로 30분 단위는 00시 00분과 00시 30분 열의 평균치로, 모든 데이터의 시간대를 1시간 단위로 일치시키고, 결측치 및 25시 이후 예외 데이터 정제.
•	평일로 한정: ‘혼잡도’와 ‘노인 승하차 수’의 데이터는 주말 데이터를 포함하고 있으나, 현재 집중하고자 하는 것은 평일 노인의 지하철 이용과 관련된 것이므로 주말 데이터는 삭제. 
•	역세권 텍스트 도출: 역주변 데이터에서 교통약자와 밀접한 키워드(예: 병원, 보건소, 복지관, 노인정, 공원 등)를 필터링하여 노인 무임승차 제한 정책 의사결정 시 예외로 두어야 할 역 데이터를 산출.

📊 시각화 결과
1️⃣ **Tab 1 요구사항 (교통 안전 및 복지 최적화)**
사용자가 '역명' 또는 '호선'을 선택하면, 해당 역의 **시간대별 노인 승하차 인원**과 **전체 열차 혼잡도**를 한눈에 비교할 수 있는 이중 축 그래프(Line/Bar Chart)

💡 인사이트: 사용자가 선택한 자하철역의 시간대별 고령층 이용자 수를 분석합니다. 피크 시간대 무임승차 혜택의 일시적 제한을 위한 근거가 됩니다.

2️⃣ **Tab 2 요구사항 (역세권 시설 Top 5 분석)**
`역세권.csv`와 `노인이용.csv`를 결합하여, 노인 이용객 수 기준 상위 5개 역사 주변에 어떤 주요 인프라/시설(`역주변` 컬럼)이 밀집해 있는지 요약

💡 인사이트: 병원, 공원, 복지관 등 키워드의 고령층 총이용량을 통해 규체 검토 대상을 도출합니다.

3️⃣ **Tab 3 요구사항 (역세권 특성별 혼잡도 분석 및 AI 리포트)**
왼쪽 컬럼: `역세권.csv`에 나타난 인프라 특징(상업형, 주거형, 대학가 등)에 따라 시간대별 평균 혼잡도 패턴을 분석하여 `st.line_chart`로 시각화
오른쪽 컬럼: 특정 역을 선택하고 'AI 분석 리포트 생성' 버튼을 누르면, Gemini API (`gemini-2.5-flash`)를 호출하여 해당 역의 역세권 특징과 혼잡도 데이터를 기반으로 "고령층 안전 대책 및 교통 복지 개선 방안에 대한 3줄 요약 리포트"를 생성해 화면에 출력

💡 인사이트: 현황 분석 및 정책 필요성, 무임승차 시간 제한 정책 제안 내용을 담은 보고서가 실시간 AI 를 통해 만들어집니다.
